### 1. PORT=3001
**What it does:** Tells your backend server which port to run on
**Why you need it:** Every server needs a port number
**For mobile app:** Your app connects to https://samaj-app-api.onrender.com:3001/api
**Can you remove it?** ‚ùå NO - Backend won't know which port to use

### 2. JWT_SECRET=BKJxvzGiVHLfS7Ik1syciq2+qKrxEdQX+h4FllE/R6g=
**What it does:** Secret key to encrypt/decrypt login tokens
**Why you need it:** When user logs in, backend creates a token for security
**For mobile app:** App stores this token, sends it with every request
**Can you remove it?** ‚ùå NO - Login/authentication won't work

---

### 3. FRONTEND_URL=http://localhost:8081
**What it does:** Tells backend which apps are allowed to connect (CORS security)
**Why you need it:** Prevents random websites from using your API

**Development (Local testing):**
```
FRONTEND_URL=http://localhost:8081
```
- When you test app on your computer
- Expo runs on localhost:8081
- Backend allows this origin

**Production (After deploying to stores):**
```
FRONTEND_URL=*
```
OR
```
FRONTEND_URL=https://samaj-app-a3de3.firebaseapp.com
```
- Mobile apps don't have a "URL" like websites
- Use * to allow all origins (mobile apps)
- Or use your Firebase Auth domain

**Can you remove it?** ‚ùå NO - CORS will block your mobile app!

---

### 4. ENABLE_ADMIN_BOOTSTRAP=true
**What it does:** Auto-creates admin account when server starts
**Why you need it:** So you have an admin login to manage app
**For mobile app:** Admin can login and manage content from admin panel
**Can you remove it?** ‚úÖ YES (after first admin is created)
- Set to `false` after first deployment
- Admin account already exists in database

---

### 5. FIREBASE_PROJECT_ID, FIREBASE_CLIENT_EMAIL, FIREBASE_PRIVATE_KEY
**What they do:** Connect backend to Firebase/Firestore database
**Why you need them:** Backend reads/writes data to Firestore
**For mobile app:** When app requests data, backend fetches from Firestore
**Can you remove them?** ‚ùå NO - Backend can't access database!

---

### 6. ADMIN_EMAIL, ADMIN_PASSWORD, ADMIN_NAME, ADMIN_MEMBER_ID
**What they do:** Default admin credentials for first-time setup
**Why you need them:** Creates admin user automatically
**For mobile app:** You login with these credentials as admin
**Can you remove them?** ‚úÖ YES (after admin is created)
- Only needed for bootstrap
- After deployment, delete these for security

ü§î WHY FRONTEND_URL IF NO WEB FRONTEND?
================================================================================

**FRONTEND_URL ‚â† Web hosting!**

It's for **CORS (Cross-Origin Resource Security)**:

### What is CORS?
When your mobile app tries to connect to backend:
1. Mobile app: "Hey backend, can I fetch data?"
2. Backend: "What's your origin?"
3. Mobile app: "I'm from expo://..."
4. Backend: "Is expo:// in my FRONTEND_URL list?"
   - ‚úÖ Yes ‚Üí Allow request
   - ‚ùå No ‚Üí Block request (CORS error)

### Why it exists:
**Security!** Prevents random apps/websites from stealing your API.

### For mobile apps:
Mobile apps don't have a traditional "origin" like websites.

**Solution 1:** Allow all origins
```env
FRONTEND_URL=*
```

**Solution 2:** Use Firebase Auth domain
```env
FRONTEND_URL=https://samaj-app-a3de3.firebaseapp.com
```

**Solution 3:** Allow specific origins (more secure)
```env
FRONTEND_URL=http://localhost:8081,https://samaj-app-a3de3.firebaseapp.com
```

### What happens if you remove FRONTEND_URL?
Backend defaults to allowing only same origin.
Mobile app ‚Üí ‚ùå **CORS ERROR** ‚Üí Can't connect to API!

üéØ DIRECT ANSWERS TO YOUR QUESTIONS
================================================================================

Q: "Why PORT=3001?"
A: Backend server needs a port. Your mobile app connects to this port.

Q: "Why JWT_SECRET?"
A: For login security. Encrypts user tokens. Mobile app needs this to authenticate.

Q: "Why FRONTEND_URL if I'm not hosting frontend?"
A: CORS security! Tells backend "these apps are allowed to connect to me"
   - NOT for hosting
   - For security/permissions
   - Mobile apps need this to avoid CORS errors

Q: "Why ENABLE_ADMIN_BOOTSTRAP?"
A: Auto-creates admin user. You need admin login to manage mobile app content.

Q: "Can I remove anything?"
A: After deployment, you can set:
   - ENABLE_ADMIN_BOOTSTRAP=false
   - Remove ADMIN_* variables
   Everything else is REQUIRED!
